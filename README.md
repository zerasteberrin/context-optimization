# Методика оптимизации контекста запросов к агенту на базе большой языковой модели

## Intro

Появление больших языковых моделей (Large Language Models, LLMs) продемонстрировало беспрецедентные возможности в понимании естественного языка, его генерации и рассуждении. 
LLM эволюционировали от базовых систем, выполняющих инструкции, до центральных механизмов рассуждений в сложных агентных системах. 
Однако производительность и эффективность этих моделей определяется той информацией, которую они получают на этапе обучения. 
По мере усложнения решаемых задач, развивались методы проектирования и управления информацией на этапе предсказания. 
Информационный домен - это ...
Статические инструкции для LLM принято называть промптом, методы их формирования изучают в дисциплине промпт инжиниринга. 
Динамические методы формирования подсказок принято изучать в дисциплине инженерия контекста (Context Engineering). 
Эти подсказки включают в себя как простые инструктивные промпты, так и сложные внешние базы знаний. 
Применение методов КИ позволяет дополнить знания агента и направить его поведение в нужное русло. 
Цель работы - раскрыть возможности агентов в полной мере, повысить качество диалогов в заданном информационном домене.

### Формальная модель

Пусть имеется некоторый набор задач, которые необходимо решить с помощью агента, например написать код на языке python, проконсультировать покупателя или поступающего в ВУЗ. 
Множество задач включает в себя ограниченный набор двоек запрос - ответ: 
$T = {(\tau, Y^*_\tau)}$
где 
* $\tau$ - конкретный экземпляр задачи (запрос пользователя);
* $Y^*_\tau$ - условно правильный ответ на запрос. 

Вероятностная авторегрессионная языковая модель генерирует выходную последовательность $Y = (y_1, \ldots, y_T)$ путём максимизации условной вероятности:

$$
P_\theta(Y|C) = \prod_{t=1}^{T} P_\theta(y_t|y_{<t}, С(\tau)) \quad (1)
$$
где 
$С(\tau)$ - контекст, сгенерированный для задачи $\tau$;
$\theta$ - параметры языковой модели. 

В инженерии промптов контекст $C$ формируется как композиция запроса пользователя и статичной инструкции по решению задачи. 
В инженерии контекста $C$ представляет собой динамически структурированный набор информационных компонентов $c_1, c_2, \ldots, c_n$. 
Эти компоненты извлекаются из различных источников, фильтруются (отбираются по релевантности) и форматируются (приводятся к нужному виду) с помощью множества функций $F$. 

$$
c_1 = f_1(\tau, D)
c_2 = f_2(\tau, D)
\ldots
c_n = f_n(\tau, D) (2)
$$

где $D$ - набор данных, необходимый для решения набора задач $T$;
$c_i$ - различные компоненты информации:

* $c_{\text{prompt}}$ — системные инструкции и правила для извлечения и генерации контекста;
* $c_{\text{retrieve}}$ — внешние знания, извлекаемые с помощью механизмов RAG или графов знаний;
* $c_{\text{tools}}$ — описания и сигнатуры доступных внешних инструментов (вызов функций и рассуждения с интеграцией инструментов);
* $c_{\text{reason}}$ — информация из промежуточных рассуждений модели;
* $c_{\text{mem}}$ — сохраняемая информация из предыдущих взаимодействий (системы памяти и управления контекстом);
* $c_{\text{state}}$ — динамическое состояние диалога с пользователем, внешнего мира или многоагентной системы.

Конечный результат формируется за несколько итераций извлечения информации и генерации промежуточных рассуждений. 
Максимизация ожидаемого качества вывода агента формализуется как задача оптимизации. 
Пространство поиска включает в себя набор функций генерации контекста $F = \{A(F)}$.


Тогда целевая функция имеет вид:

$$
F^* = \arg\max_{F} \mathbb{E}_{\tau \sim T} \left[ \text{Reward}(P_\theta(Y|C_F(\tau)), Y^*_\tau) \right] \quad (3)
$$

* $F^*$ — оптимальный набор функций генерации контекста;
* $\tau$ — конкретный экземпляр задачи;
* $C_F(\tau)$ — контекст, сгенерированный функциями из набора $F$ для данной задачи $\tau$;
* $Y^*_\tau$ — эталонный (идеальный) результат для задачи $\tau$ (ground‑truth output);
* $P_\theta(Y|C_F(\tau))$ — распределение вероятностей выходных последовательностей $Y$, генерируемых моделью с параметрами $\theta$ при заданном контексте $C_F(\tau)$;
* $\text{Reward}(\cdot)$ — функция вознаграждения (метрика качества), оценивающая, насколько сгенерированный моделью вывод соответствует эталонному результату $Y^*_\tau$;
* $\mathbb{E}_{\tau \sim T}$ — математическое ожидание по распределению задач $T$, то есть усреднение по всем возможным задачам из рассматриваемого набора;
* $\arg\max_F$ — операция поиска такого набора функций $F$, который максимизирует среднее вознаграждение.

Эта задача оптимизации подчиняется **жёстким ограничениям**, наиболее значимое из которых — ограничение на длину контекста модели:

$$
|C| \leq L_{\text{max}}
$$

где:
* $|C|$ — длина (объём) контекста $C$ (обычно измеряется в токенах);
* $L_{\text{max}}$ — максимально допустимая длина контекста для конкретной модели (например, 4 096, 8 192 или 32 000 токенов).
